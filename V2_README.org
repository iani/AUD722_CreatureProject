** Controller for Environment-Creature coupling  

1. Controller: Environment issues changed at each state change - instead of calling methods
2. Custom behavior specification: Creatures register themselves to environment by adding a controller that translates state messages to method calls. Each Creature class has a default controller, which an instance can override.
3. Test runs: Write an event play function that calls changed on a given object, then attach a controller to a Creature on that object, and play a pattern with the custom event.
